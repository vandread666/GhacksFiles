Sommaire

Résumé

Cet article explique comment utiliser l'outil SPCheck dans Windows XP pour déterminer le numéro de Service Pack des composants installés, fichier par fichier. SPCheck.exe est fourni avec le package des outils de support de Windows XP.
Retour au début | Envoyer des commentaires

Plus d'informations

SPCheck vous permet de déterminer le numéro de Service Pack des différents composants installés sur des ordinateurs exécutant Windows XP. SPCheck génère un état qui répertorie l'origine de chaque fichier associé aux composants installés. SPCheck indique le numéro de Service Pack actuel des composants suivants :
  • Protocole Internet (TCP/IP)
  • NWLink (IPX/SPX)
  • Protocole SNMP (Simple Network Management Protocol)
  • Client pour les réseaux Microsoft

Installation de SPCheck

Pour installer SPCheck, vous devez installer les outils de support de Windows XP. Pour plus d'informations sur l'installation de ces outils, cliquez sur le numéro ci-dessous pour afficher l'article correspondant dans la Base de connaissances Microsoft.

Utilisation de SPCheck

SPCheck examine chaque composant individuellement. Il doit s'exécuter dans un contexte sécurisé équivalent au compte de l'administrateur local. Dans le cas contraire, des fichiers risquent d'être signalés comme manquants même s'ils sont installés. SPCheck recherche les fichiers à l'aide de la variable d'environnement système PATH. S'il signale des fichiers comme étant manquants (bien qu'ils soient installés), assurez-vous que les dossiers dans lesquels ces composants sont installés sont bien inclus dans la variable d'environnement système PATH.

Commutateurs de SPCheck

Vous pouvez utiliser les commutateurs suivants avec l'outil SPCheck :
  • /i : utilisez ce commutateur pour indiquer le nom du fichier .ini utilisé. Si ce commutateur est omis, l'outil utilise le fichier SPCheck.ini par défaut.
  • /r : utilisez ce commutateur pour indiquer le nom du fichier de rapport qui est généré. Si ce commutateur est omis, l'outil utilise le fichier SPCheck.rpt par défaut.

Sortie de SPCheck

Par défaut, SPCheck crée le rapport dans le dossier qui est utilisé pour exécuter SPCheck. Un état « Missing » (manquant) indique que le fichier n'a pas été trouvé sur l'ordinateur cible, tandis qu'un état « Unknown » (inconnu) indique que le fichier a été trouvé, mais qu'il ne correspond à aucun des fichiers répertoriés dans le fichier .ini. Le rapport se présente comme suit :
[XP Internet Protocol TCP/IP]
C:\WINDOWS\SYSTEM32\DRIVERS\TCPIP.SYS SP0
C:\WINDOWS\SYSTEM32\DRIVERS\MSGPC.SYS SP0-2
C:\WINDOWS\SYSTEM32\DRIVERS\NETBT.SYS Q309521
C:\WINDOWS\SYSTEM32\DRIVERS\WANARP.SYS Missing
C:\WINDOWS\SYSTEM32\DRIVERS\IPSEC.SYS Unknown

[XP Internet Protocol TCP/IP] Résumé :
Fichiers manquants : 1
Fichiers inconnus : 1
SP0 : 1
SP0-2 : 1
Q309521 : 1
Certains fichiers figurent dans plusieurs Service Packs. Par exemple, « SP0-2 » indique que le fichier est identique dans les deux premiers Service Packs. Si le Service Pack qui doit être installé sur l'ordinateur figure dans la plage indiquée pour un fichier, le fichier peut être considéré comme compatible avec ce Service Pack.

Si le Service Pack 1 (SP1) doit être installé sur l'ordinateur cible, le rapport donné en exemple indique que le SP1 figure dans les plages spécifiées pour chaque fichier. L'ordinateur cible contient des fichiers TCP/IP qui correspondent au Service Pack installé. SPCheck peut également indiquer un nombre Qnnnnnn au lieu d'un numéro de Service Pack. Ce nombre signale qu'un correctif a été installé. Pour plus d'informations sur ce correctif, interrogez la Base de connaissances Microsoft en précisant ce numéro d'article TechNet.

Notez que SPCheck ne détecte qu'un petit nombre de correctifs. Par conséquent, des fichiers de correctifs peuvent être signalés comme étant inconnus.

L'outil SPVerify.exe permet de simuler l'installation d'un Service Pack ou d'un correctif Windows et d'obtenir d'autres informations se rapportant à un Service Pack ou à un correctif Windows. Pour plus d'informations sur l'outil SPVerify, cliquez sur le numéro ci-dessous pour afficher l'article correspondant dans la Base de connaissances Microsoft.
