PROGRAM FEATURES

OE5 and OE6 have a proprietary binary format that makes archiving individual messages difficult. In addition, a number of people have indicated that their message folders have become corrupted and OE can not read them. Consequently, they have lost all the messages in that folder.

DBXtract extracts all mail and news messages from individual dbx files. After extracting the messages one can drag them from a Windows Explorer folder into an Outlook Express mail folder.

It requires the existence of the VB6 runtime dll, msvbvm60.dll. If you do not have that in your windows\system (or system32 for NT / 2000) directory, you can download it directly from Microsoft, by CLICKING HERE .

The program runs in Win9x, WinMe, NT, Windows 2000, WindowsXP, and Windows Server 2003.

After installing the VB6 runtime files, you can then run DBXtract.

New to Version 5

1. One can select multiple dbx files from which to extract.

2. One can extract to subdirectories named after the file from which one is extracting. For instance, if extracting from the inbox, the extracted files are stored under a subdirectory called "Inbox".

KNOWN PROBLEMS

DBXtract will not function properly with Asian versions of Windows. If your dbx file is copied to a machine that has a non-Asian version of Windows, then you can use DBXtract on that machine to extract your messages. Then you can copy the extracted messages back to the machine with the Asian version of Windows and then reincorporate the messages into OE. You also might try changing the user interface from Asian to English and see if that fixes the problem. Alternatively, one can try installing a Unicode character set instead of a DBCS (double byte character set).

If you get a message with "Error 75" during extraction, then the dbx file is likely cross-linked and you can either try making a copy of the file and running DBXtract on that, or run scandisk first and then try DBXtract again.
